1. 准备环境

回顶部
官方 Quick Start 结论： 先装 Docker Desktop 即可。AI Provider 的 API Key 不用一开始就写进 compose。

建议目录

比如新建一个独立文件夹并进入，然后可以下载yml：open-notebook-docker

mkdir open-notebook-docker
cd open-notebook-docker
curl -o docker-compose.yml https://raw.githubusercontent.com/lfnovo/open-notebook/main/docker-compose.yml

容器角色

  • surrealdb：数据库
  • open_notebook：应用本体
  • ./surreal_data：数据库持久化
  • ./notebook_data：Open Notebook 数据持久化

2. 如果没执行上述的CURL命令，直接写入 docker-compose.yml

回顶部

下面是基于官方 README 整理的最小可用配置。直接复制保存成 docker-compose.yml。

docker-compose.yml 
services:
  surrealdb:
    image: surrealdb/surrealdb:v2
    command: start --log info --user root --pass root rocksdb:/mydata/mydatabase.db
    user: root
    ports:
      - "8000:8000"
    volumes:
      - ./surreal_data:/mydata
    restart: always

  open_notebook:
    image: lfnovo/open_notebook:v1-latest
    ports:
      - "8502:8502"
      - "5055:5055"
    environment:
      - OPEN_NOTEBOOK_ENCRYPTION_KEY=change-me-to-a-secret-string
      - SURREAL_URL=ws://surrealdb:8000/rpc
      - SURREAL_USER=root
      - SURREAL_PASSWORD=root
      - SURREAL_NAMESPACE=open_notebook
      - SURREAL_DATABASE=open_notebook
    volumes:
      - ./notebook_data:/app/data
    depends_on:
      - surrealdb
    restart: always
必须改： OPEN_NOTEBOOK_ENCRYPTION_KEY 不能继续用示例值。换成你自己的长随机字符串。如my-secret-321123

关键环境变量

  • OPEN_NOTEBOOK_ENCRYPTION_KEY：应用加密密钥
  • SURREAL_URL：容器内数据库地址
  • SURREAL_NAMESPACE / SURREAL_DATABASE：数据库命名空间

端口说明

  • 8502：前端主访问地址
  • 5055：应用辅助接口端口
  • 8000：SurrealDB 端口

3. 启动服务

回顶部

在 `docker-compose.yml` 所在目录执行：

启动命令 
docker compose up -d

会等待一段时间，下载镜像Images,然后检查状态：

状态检查 
docker compose ps
docker compose logs -f open_notebook
  1. 先等 15-20 秒。下载完后会在container中运行，我们进入container查看状态,container会显示这个应用的端口分布，点击
  2. 浏览器打开 http://localhost:8502。
  3. 如果打不开，先看 `docker compose ps` 是否两个服务都为 running。
  4. 再看日志里是否存在端口占用、数据库连接失败、加密 key 配置问题。
首次启动慢一点正常。 Docker 会先拉镜像，再初始化数据库与应用。

4. 配置 AI Provider

回顶部

官方 README 说得很明确：API Key 在 UI 里后配。进入应用后走这条路径：

  1. 打开 Settings → API Keys
  2. 点击 Add Credential
  3. 选择 Provider：OpenAI / Anthropic / Google / Ollama / Deepseek
  4. 粘贴 API Key 并保存
  5. 点击 Test Connection
  6. 点击 Discover Models → Register Models
  7. 点击 embeded Models 时，我们发现一个问题，deepseek-v4只提供了language模型，如果要让整个应用跑起来，需要配置相应的embeded模型，如gpt-4o ，我们需要处理一些数据

云模型

最省事。OpenAI、Anthropic、Google、Groq 等都能配。

本地模型

如果你想省 API 费用，可后续接 Ollama。官方 README 也提到 examples/docker-compose-ollama.yml。

5. 常见问题

回顶部

Q1. `localhost:8502` 打不开

先执行 docker compose ps。如果容器没起来，再看日志：

docker compose logs -f surrealdb
docker compose logs -f open_notebook

Q2. 提示加密密钥问题

说明 OPEN_NOTEBOOK_ENCRYPTION_KEY 没改或格式有问题。 直接换成更长、随机、不可预测的字符串，再重启容器。

Q3. 端口冲突

改 compose 左侧端口映射，比如把 `8502:8502` 改成 `18502:8502`。

Q4. Windows 下卷权限异常

确认 Docker Desktop 已允许当前磁盘共享。必要时删掉失败容器， 保留 compose 文件后重新 `docker compose up -d`。

Q5. 配完 API Key 仍找不到模型

再点一次 Test Connection、Discover Models、 Register Models。部分 Provider 还要检查区域或账号权限。

6. 这个教程站总结

回顶部

  • 这个教程主要是为了介绍Docker部署的步骤和注意事项，内容相对简洁，适合快速上手。后续考虑增加一些二次开发的内容，比如如何在这个基础上接入更多的模型、如何做数据备份和迁移、如何监控容器状态等。
  • Open Notebook 的灵活性和可扩展性也很适合做一些定制化的开发，比如接入企业内部的知识库、做一些特定领域的微调等。总之，这个教程站的目标是让更多的中文用户能够快速上手 Open Notebook 的 Docker 部署，并且在这个基础上探索更多的可能性。
  • 我们主要是为了替代notebookLM，提供一个开源、可自托管的解决方案。通过这个教程，希望能够降低部署的门槛，让更多人能够体验到 Open Notebook 的强大功能，同时也欢迎大家在 GitHub 上提出问题和贡献代码，一起完善这个项目。比如国内用户最喜欢的deepseek也加入这个项目，给了大家探索空间
  • Docker让我们简化了复杂应用的各种配置，直接运行的效率显著提高，更快吸引用户。有点像steam会为各种游戏配置依赖，而不像过去的玩家缺失配置文件需要到处寻找，仿佛专业的人才能安装和玩游戏
  • 我们的研究灵感来自员工需要使用notebookLM，同时对视频和音频的AI分析有一定的要求，所以决定研究open-notebook
